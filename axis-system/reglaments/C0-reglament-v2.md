# РЕГЛАМЕНТ C0 — Приём и классификация входящих задач

**AXIS | Дизайн-бюро архитектурных решений**

Версия: 2.0 | Дата: 2026-02-16
Методология: Академия Системного Бизнеса, Модуль 3 «Регламенты и инструкции»
Владелец процесса: Главный дизайнер
Тип регламента: Простой (один исполнитель)

---

## 1. ЦЕЛЬ ПРОЦЕССА

Обеспечить фиксацию, классификацию и маршрутизацию 100% входящих задач от заказчика.

**Исключить:**
- Потерю задач
- Выполнение работ вне зоны ответственности
- «Устные» договорённости без фиксации

---

## 2. ЦКП — ЦЕННЫЙ КОНЕЧНЫЙ ПРОДУКТ

**ЦКП процесса:** Каждая входящая задача зафиксирована в Trello, классифицирована по категории (🟢🟡🔴🔵), имеет ответственного и дедлайн. Задачи вне зоны ответственности возвращены заказчику с указанием маршрута.

**Критерий достижения ЦКП:**
- [ ] Карточка создана
- [ ] Категория определена (метка проставлена)
- [ ] Обязательные поля заполнены
- [ ] Для 🟢 — передано на целевую доску
- [ ] Для 🔴 — возвращено с указанием кому

---

## 3. ВХОДЫ И ВЫХОДЫ

| Вход | Источник | Формат |
|------|----------|--------|

## 5. RACI-МАТРИЦА

| Функция | Главный дизайнер | Заказчик | Архитекторы |
|---------|:----------------:|:--------:|:-----------:|
| Приём задачи | **R/A** | I | — |
| Классификация | **R/A** | — | — |
| Сбор исходников | **R/A** | C | I |
| Маршрутизация | **R/A** | I | I |
| Ответ на консультацию | **R/A** | I | — |
| Возврат задачи | **R/A** | I | — |
| Предложение допуслуги | **R/A** | C | — |

**Легенда:**
- **R** = Responsible (кто делает)
- **A** = Accountable (кто отвечает)
- **C** = Consulted (с кем согласовывают)
- **I** = Informed (кого информируют)

## 6. АЛГОРИТМ ВЫПОЛНЕНИЯ

### Шаг 1: Фиксация

**Триггер:** Любое обращение от заказчика (звонок, сообщение, встреча)

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 1.1 | Получить задачу от заказчика | Задача услышана/прочитана |
| 1.2 | Создать карточку на доске «Входящие» → «Новое» | Карточка создана |
| 1.3 | Заполнить обязательные поля (см. п. 6.5) | Все поля заполнены |

**Срок:**
- Рабочее время (09:00–19:00): 15 минут
- Нерабочее время: до 10:00 следующего рабочего дня
- На встрече: сразу после завершения встречи

**Правило:** Нет карточки = задачи не существует

---

### Шаг 2: Классификация

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 2.1 | Определить категорию по Матрице ответственности | Категория определена |
| 2.2 | Поставить метку на карточку | Метка проставлена |

**Таблица классификации:**

| Метка | Категория | Примеры | Что делать |
|:-----:|-----------|---------|------------|
| 🟢 | Профильная | Концепция, визуализация, рабочка, авторский надзор | Маршрутизировать → Шаг 3A |
| 🟡 | Консультация | Согласование места кондиционера, разъяснение проекта | Ответить → Шаг 3B |
| 🔴 | Не моё | Бухгалтерия, снабжение, логистика, контроль монтажа | Вернуть → Шаг 3C |
| 🔵 | Допуслуга | Координация подрядчиков, пересадка деревьев | Предложить → Шаг 3D |

**При сомнении:** Открыть Матрицу ответственности. Если задачи нет ни в одной зоне AXIS — это 🔴 «Не моё».

---

### Шаг 3A: Маршрутизация (для 🟢 Профильных)

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 3A.1 | Определить процесс: C1/C2/C3/C4/C5 | Процесс определён |
| 3A.2 | Проверить чеклист приёмки (п. 6.6) | Все данные собраны |
| 3A.3 | Если данных нет → запросить у заказчика, карточка в «Ожидает ответа» | Запрос отправлен |
| 3A.4 | Создать карточку на целевой доске (C1–C5) | Карточка создана |
| 3A.5 | Добавить ссылку на новую карточку в комментарий | Ссылка добавлена |
| 3A.6 | Переместить карточку в «Передано в работу» | Статус изменён |

**ЦКП шага:** Карточка на целевой доске, ссылка в комментарии

---

### Шаг 3B: Ответ и закрытие (для 🟡 Консультаций)

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 3B.1 | Ответить на вопрос заказчика | Ответ дан |
| 3B.2 | Зафиксировать решение в комментарии к карточке | Решение задокументировано |
| 3B.3 | Переместить карточку в «Закрыто» | Статус изменён |

**Формат комментария:**
```
[Дата] Решение: {что решили}
Кто: Главный дизайнер
```

**ЦКП шага:** Ответ дан, решение зафиксировано, карточка закрыта

---

### Шаг 3C: Возврат задачи (для 🔴 Не моё)

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 3C.1 | Определить кому перенаправить по Матрице ответственности | Адресат определён |
| 3C.2 | Сообщить заказчику что задача вне зоны ответственности | Сообщение отправлено |
| 3C.3 | Указать кому адресовать | Адресат назван |
| 3C.4 | Зафиксировать в карточке: кому, дата | Информация записана |
| 3C.5 | Переместить карточку в «Возвращено заказчику» | Статус изменён |

**Шаблон ответа:**
```
{Имя}, эта задача относится к зоне ответственности {роль}.

По нашему договору AXIS отвечает за дизайн, визуализацию, 
рабочую документацию и авторский надзор.

Рекомендую адресовать это {кому}.
```

**ЦКП шага:** Заказчик уведомлён, задача перенаправлена

---

### Шаг 3D: Предложение допуслуги (для 🔵)

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 3D.1 | Сообщить что задача вне договора | Сообщение отправлено |
| 3D.2 | Предложить выполнение как допуслугу | Предложение озвучено |
| 3D.3 | Если согласен — согласовать стоимость и сроки | Условия зафиксированы |
| 3D.4 | Если не согласен — вернуть как 🔴 | См. Шаг 3C |

**ЦКП шага:** Либо допуслуга согласована, либо задача возвращена

---

### Шаг 4: Обработка изменений

**Триггер:** Заказчик меняет задачу после приёма в работу

**Действия:**

| # | Действие | Критерий выполнения |
|---|----------|---------------------|
| 4.1 | Зафиксировать изменение в комментарии: дата, что изменилось, кто попросил | Изменение задокументировано |
| 4.2 | Оценить влияние на сроки и объём | Оценка сделана |
| 4.3 | Если существенно — согласовать новый дедлайн с заказчиком | Дедлайн обновлён |
| 4.4 | Если меняет суть — закрыть старую карточку, создать новую | Новая карточка создана |

**Правило:** Каждое изменение = комментарий с датой

| Контакт | Ответственный у заказчика | `Иван, +7-XXX-XXX-XX-XX` |
| Приоритет | 🔴 Горит / 🟡 Средний / 🟢 Обычный | 🟡 Средний |

### 6.6 Чеклист приёмки (для 🟢 Профильных)

Перед созданием карточки на целевой доске проверить:

- [ ] Адрес объекта
- [ ] Тип помещения / территории
- [ ] Площадь
- [ ] Контакт ответственного на стороне заказчика
- [ ] Характер задачи (что именно нужно сделать)
- [ ] Планировки / разрезы / исходные данные (если применимо)
- [ ] Дедлайн согласован

**Правило:** Не передавать задачу в работу без полного комплекта данных.
