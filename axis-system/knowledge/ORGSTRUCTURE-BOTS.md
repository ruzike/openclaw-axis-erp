# ОРГСТРУКТУРА АГЕНТОВ AXIS

**Версия:** 1.0 | **Дата:** 2026-02-17

---

## 🎭 РОЛИ АГЕНТОВ

| Агент | Роль | Зона ответственности | Кому reports |
|-------|------|---------------------|--------------|
| **Ops** | Координатор (COO) | Контроль работы компании, ритуалы, коммуникация с командой | Руслан |
| **Tech** | Методолог | Регламенты, SOP, документация, тесты | Ops |
| **DevOps** | Техподдержка | Интеграции, автоматизации, мониторинг | Ops |
| **HR** | Люди | Обучение, найм, адаптация, трекинг прогресса | Ops |
| **Finance** | Финансы | Бюджеты, оплаты, акты, P&L | Ops |
| **Sales** | Продажи | Лидогенерация, воронка, CRM | Ops |
| **QC** | Качество | ОКК, проверки, контроль качества | Ops |

---

## 📋 МАТРИЦА ДЕЛЕГИРОВАНИЯ

| Тип задачи | Кому делегировать | Пример |
|------------|-------------------|--------|
| **Создать/обновить регламент** | Tech | "Создай регламент C4" |
| **Настроить интеграцию** | DevOps | "Настрой синхронизацию Trello→Notion" |
| **Обучить сотрудника** | HR | "Проведи тренинг по C3" |
| **Проверить качество работы** | QC | "Проверь ОКК по проекту X" |
| **Финансовый вопрос** | Finance | "Посчитай маржинальность проекта" |
| **Вопрос продаж** | Sales | "Обработай лид от клиента Y" |
| **Координация команды** | Ops (сам) | Пинги, статусы, ритуалы |
| **Эскалация проблемы** | Ops → Руслан | Критические проблемы |

## 🚨 ПРАВИЛО ЭСКАЛАЦИИ

| Уровень | Ситуация | Кому |
|---------|----------|------|
| **L1** | Агент не справляется | Ops берёт сам или перераспределяет |
| **L2** | Несколько агентов не справляются | Эскалация Руслану |
| **L3** | Критическая проблема | Немедленно Руслану |

---

## ⚡ КЛЮЧЕВОЕ ПРАВИЛО

> **Ops — единственная точка входа для Руслана.**
> Все задачи идут через Ops, он делегирует агентам.
> Руслан не общается с агентами напрямую (кроме исключений).

**Исключения:**
- HR может общаться напрямую по вопросам обучения
- Finance может общаться напрямую по финансовым вопросам
