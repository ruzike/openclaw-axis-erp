# Промпт для OpenClaw.ai бота: Настройка Trello-доски для процесса C3

## Контекст
Ты настраиваешь доску Trello для дизайн-бюро архитектурных решений OpenClaw.ai.
Доска управляет процессом C3 «Рабочий проект (рабочка)» — производство комплекта рабочих чертежей.

Методология: Академия Системного Бизнеса (АСБ), Модуль 3 «Регламенты и инструкции».

Принципы АСБ для Trello:
- Карточка = единый источник правды по задаче (никакие задачи не передаются устно)
- Чеклист в карточке = критерий передачи ЦКП (ценного конечного продукта)
- Колонки = статусы по PDCA-циклу (Plan → Do → Check → Act)
- Перемещение карточки = формальная передача ЦКП между участниками

Участники: Главный дизайнер (владелец процесса), Архитектор 1, Архитектор 2.
Типы проектов: Холл ЖК, Квартира, Благоустройство.

**Название доски:** `C3 — Рабочий проект`

**Описание доски:**
```
Процесс C3: Производство рабочих чертежей.
Владелец процесса: Главный дизайнер.
Регламент: Регламент C3 v1.0.

Правила:
• Каждый проект = 1 карточка
• Задача передаётся ТОЛЬКО через карточку (не устно)
• Правки фиксируются в файле, не устно
• Проект сдан когда ВСЕ пункты чеклиста отмечены
```

```

- **Тип:** Холл ЖК
- **Объект:** {Название ЖК}
- **Исполнитель:** {Архитектор}
- **Дедлайн:** {дата}
- **Ссылка на папку:** {ссылка}

1. Главный дизайнер готовит исходники → папка 01_Исходники
2. Архитектор чертит по порядку: планы → развёртки → ведомости
```

## Задача 5: Настроить автоматизации (Butler)

Если доступен Trello Butler, настрой следующие правила:

### Автоматизация 1: Напоминание о дедлайне
```
Триггер: за 2 дня до дедлайна карточки
Действие: отправить уведомление всем участникам карточки
Текст: "⏰ Дедлайн через 2 дня. Проверьте статус чеклиста."
```

### Автоматизация 2: Просроченный дедлайн
```
Триггер: дедлайн карточки наступил
Действие: добавить красную метку "ПРОСРОЧЕНО" на карточку
```

### Автоматизация 3: Уведомление владельца при переносе в «На проверку»
```
Триггер: карточка перемещена в колонку «На проверку»
Действие: добавить комментарий "@Главный дизайнер — альбом готов к проверке по чеклисту ОКК"
```

### Автоматизация 4: Метрика — дата завершения
```
Триггер: карточка перемещена в колонку «Готово»
Действие: добавить комментарий "✅ Проект завершён: {дата}. Замерить метрики: % ОКК с 1 раза, количество итераций правок, время от старта до готово."
```

## Задача 6: Создать пилотную карточку

Создай одну реальную карточку в колонке **Бэклог** на основе ближайшего шаблона — чтобы можно было сразу запустить пилотный проект по регламенту C3.

**Название:** `[ПИЛОТ] {Тип} — {Объект}`

В описании добавить:
```
🧪 ПИЛОТНЫЙ ПРОЕКТ по Регламенту C3 v1.0
Цель: проверить регламент на практике и замерить метрики.

Метрики для замера:
• % чеклиста ОКК пройденный с 1-го раза
• Количество итераций правок до «Готово»
• Время от старта до «Готово»
• Топ-3 ошибки по чеклисту ОКК
```
