# Промпт для OpenClaw.ai бота: Настройка Trello-доски для процесса C1+C2

## Контекст
Ты настраиваешь объединённую доску Trello для дизайн-бюро архитектурных решений OpenClaw.ai.
Доска управляет процессом C1 (Концепция) и C2 (Визуализация) — дизайн-проектирование и создание 3D-визуализации.

Методология: Академия Системного Бизнеса (АСБ), Модуль 3 «Регламенты и инструкции».

Принципы АСБ для Trello:
- Карточка = единый источник правды по задаче (никакие задачи не передаются устно)
- Чеклисты этапов = критерий перехода между этапами
- Колонки = статусы по процессу (от идеи до готового альбома)
- Перемещение карточки = формальная передача ЦКП между этапами

Участники: Главный дизайнер (владелец процесса), Архитектор, Визуализатор, Заказчик.
Типы проектов: Концепция (C1), Визуализация (C2).

---

## Задача 1: Создать доску

**Название доски:** `C1+C2 — Концепция и Визуализация`

**Описание доски:**
```
Процесс C1+C2: Разработка концепции и создание 3D-визуализации.
Владелец процесса: Главный дизайнер.
Регламент: Регламент C1 v1.0 + Регламент C2 v1.0.

Правила:
• Задача из C0 → Бэклог → Очередь → В работе → На проверку → Правки → Готово
• C1 и C2 часто идут последовательно на одном объекте
• Лимит правок: 2 круга (более — почасовая оплата)
• Каждый этап = комментарий в Trello с датой и результатами
```

## Задача 2: Создать 6 колонок (списков)

Создай колонки строго в этом порядке слева направо:

| # | Название колонки | Описание (добавить в описание списка если возможно) |
|---|------------------|------------------------------------------------------|
| 1 | **Бэклог** | Задачи получены из C0, но исходные данные ещё не подготовлены |
| 2 | **Очередь** | Задачи на этой неделе, назначены сроки |
| 3 | **В работе** | Архитектор/Визуализатор выполняет этапы |
| 4 | **На проверку** | Этап завершён — ждёт проверки Главного дизайнера |
| 5 | **Правки** | Есть замечания, файлы в папке с правками |
| 6 | **Готово** | ЦКП достигнут, альбом отправлен заказчику |

## Задача 4: Создать 2 карточки-шаблона

Для каждого типа процесса создай карточку-шаблон в колонке **Бэклог**.

### Шаблон 1: Концепция (C1)

**Название карточки:** `[ШАБЛОН] Концепция — {Название объекта}`

**Метка:** C1 — Концепция (синяя)

**Описание карточки:**
```
## Информация о проекте
- **Тип:** Концепция (C1)
- **Объект:** {Название ЖК / Квартира / Территория}
- **Исполнитель:** {Визуализатор / Архитектор}
- **Дедлайн:** {дата}
- **Ссылка на папку:** {ссылка}

## Процесс (по регламенту C1)
1. Главный дизайнер анализирует ТЗ, собирает референсы → Мудборд
2. Согласование стиля с Заказчиком (Мудборд)
3. Визуализатор строит «коробку» (White Box) по чертежам
4. Согласование ракурсов с Главным дизайнером
5. Визуализатор находит материалы, делает рендеры
6. Главный дизайнер собирает Альбом презентации
7. Презентация Заказчику → Сбор правок
8. Визуализатор вносит правки → Финальный Альбом

## Правила
⚠ Не начинать работу без полного комплекта данных (обмеры, ТЗ)
⚠ Мудборд нужно согласовать с Заказчиком до начала White Box
⚠ Лимит правок: 2 круга (более — почасовая оплата)
⚠ Каждый этап = комментарий в Trello с датой и результатами
```

**Чеклист 1 — «Исходные данные»:**
```
☐ ТЗ полностью изучен
☐ Обмерный план получен и проанализирован
☐ Фотофиксация объекта есть
☐ Референсы подобраны (10-15 картинок)
☐ Мудборд создан и согласован
☐ Ракурсы согласованы
```

**Чеклист 2 — «White Box (геометрия)»:**
```
☐ «Коробка» помещения/здания построена в 3D
☐ Условная мебель расставлена согласно планировке
☐ Свет и камеры настроены
☐ Черновые рендеры (Clay) готовы (3-4 картинки)
☐ Ракурсы согласованы с Главным дизайнером
```

**Чеклист 3 — «Детализация и рендеры»:**
```
☐ Материалы настроены согласно Мудборду
☐ Сцена наполнена деталями (декор, люди, зелень)
☐ Финальные рендеры (High Quality) готовы
☐ Пост-обработка в Photoshop выполнена
```

**Чеклист 4 — «Альбом и утверждение»:**
```
☐ Альбом презентации сверстан (Обложка, Пояснение, Планировка, Рендеры)
☐ Альбом отправлен Заказчику (или проведена презентация)
☐ Получен список правок (единым списком!)
☐ Правки внесены (если есть)
☐ Получено финальное утверждение (подпись/сообщение)
☐ Карточка перемещена в «Готово»
```

После создания — **пометить карточку как шаблон** (Card Template).

**Метка:** C2 — Визуализация (фиолетовая)

**Описание карточки:**
```

⚠ Ракурсы нужно согласовать до создания рабочих рендеров
```

## Задача 5: Настроить автоматизации (Butler)

Если доступен Trello Butler, настрой следующие правила:

### Автоматизация 1: Напоминание о дедлайне
```
Триггер: за 1 день до дедлайна карточки
Действие: добавить комментарий "@ruslan_archviz — ⏰ Дедлайн через 1 день. Проверьте статус."
```

### Автоматизация 2: Просроченный дедлайн
```
Триггер: дедлайн карточки наступил
Действие: добавить красную метку "ПРОСРОЧЕНО"
```

### Автоматизация 3: Уведомление при переносе в «Правки»
```
Триггер: Карточка перемещена в «Правки»
Действие: добавить комментарий "📝 Правки: {количество замечаний}"
```

### Автоматизация 4: Уведомление при переносе в «Готово»
```
Триггер: Карточка перемещена в «Готово»
Действие: добавить комментарий "✅ Концепция/Визуализация завершена: {дата}"
```

### Автоматизация 5: Метрика — количество итераций
```
Триггер: Карточка перемещена в «Правки»
Действие: добавить комментарий "Итерация правок: {количество}. Максимум: 2."
```
