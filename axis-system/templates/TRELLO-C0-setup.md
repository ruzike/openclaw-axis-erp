# Промпт для OpenClaw.ai бота: Настройка Trello-доски для процесса C0

## Контекст
Ты настраиваешь доску Trello для дизайн-бюро архитектурных решений OpenClaw.ai.
Доска управляет процессом C0 «Входящие задачи» — приём и классификация всех обращений от заказчика.

Методология: Академия Системного Бизнеса (АСБ), Модуль 3 «Регламенты и инструкции».

Принципы АСБ для Trello:
- Карточка = единый источник правды по задаче (никакие задачи не передаются устно)
- Чеклист приёмки = критерий готовности к передаче на целевую доску
- Колонки = статусы по процессу (от приёма до закрытия)
- Перемещение карточки = формальная передача ЦКП между участниками

Участники: Главный дизайнер (владелец процесса), Заказчик, Архитекторы.
Типы задач: Профильные (C1-C5), Консультации, Не мои, Допуслуги.

---

## Задача 1: Создать доску

**Название доски:** `C0 — Входящие`

**Описание доски:**
```
Процесс C0: Приём и классификация входящих задач.
Владелец процесса: Главный дизайнер.
Регламент: Регламент C0 v2.0.

Правила:
• Каждый входящий = карточка в Trello (никакие задачи не передаются устно)
• Классификация по Матрице ответственности (🟢🟡🔴🔵)
• Обязательные поля карточки
• Задача передаётся на целевую доску ТОЛЬКО после полного комплекта данных
```

---

## Задача 2: Создать 6 колонок (списков)

Создай колонки строго в этом порядке слева направо:

| # | Название колонки | Описание (добавить в описание списка если возможно) |
|---|------------------|------------------------------------------------------|
| 1 | **Новое** | Задача зафиксирована, не классифицирована |
| 2 | **В обработке** | Классифицирована, собираются данные |
| 3 | **Передано в работу** | 🟢 Профильная — на доске C1–C5 |
| 4 | **Ожидает ответа** | Ждём данные от заказчика |
| 5 | **Возвращено заказчику** | 🔴 Не моё — перенаправлено |
| 6 | **Закрыто** | 🟡 Консультация завершена или отменена |

## Задача 3: Создать 4 метки (labels)

## Задача 4: Создать 4 карточки-шаблона (для каждой категории)

Для каждой категории задач создай карточку-шаблон в колонке **Новое**.

### Шаблон 1: Профильная (C1–C5)

**Название карточки:** `[ШАБЛОН] Профильная — {Тип процесса} — {Название объекта}`

**Метка:** Профильная (зелёная)

**Описание карточки:**
```
## Информация о задаче
- **Тип процесса:** {C1/C2/C3/C5}
- **Объект:** {Название ЖК / Квартира / Территория}
- **Исполнитель:** {Архитектор / Визуализатор}
- **Дедлайн:** {дата}
- **Ссылка на папку:** {ссылка}

## Процесс (по регламенту C0)
1. Проверить полноту исходных данных (обмеры, фото, ТЗ)
2. Заполнить чеклист приёмки (п. 6.6 регламента C0)
3. Если данные неполные — переместить в «Ожидает ответа»
4. Если данные полные — создать карточку на целевой доске (C1–C5)
5. Добавить ссылку на новую карточку в комментарий
6. Переместить карточку в «Передано в работу»

## Правила
⚠ Не начинать работу без полного комплекта исходных данных
⚠ Каждое изменение = комментарий с датой
⚠ Задача передаётся ТОЛЬКО через карточку, не устно
```

**Чеклист приёмки (п. 6.6 регламента C0):**
```
☐ Адрес объекта
☐ Тип помещения / территории
☐ Площадь
☐ Контакт ответственного на стороне заказчика
☐ Характер задачи (что именно нужно сделать)
☐ Планировки / разрезы / исходные данные (если применимо)
☐ Дедлайн согласован
```

**Чеклист перехода на целевую доску:**
```
☐ Чеклист приёмки отмечен (все пункты)
☐ Обязательные поля заполнены
☐ Метка категории проставлена
☐ Ссылка на целевую доску добавлена в комментарий
```

После создания — **пометить карточку как шаблон** (Card Template).

### Шаблон 2: Консультация

**Название карточки:** `[ШАБЛОН] Консультация — {Тема вопроса}`

**Метка:** Консультация (жёлтая)

**Описание карточки:**
```

- **Тип:** Консультация
- **Тема:** {Краткое описание вопроса}

- **Дата поступления:** {дата}

1. Ответить на вопрос заказчика
2. Зафиксировать решение в комментарии
3. Переместить карточку в «Закрыто»

## Правила
⚠ Ответить в течение 24 часов
⚠ Решение зафиксировать в комментарии (дата, что решили, кто)
⚠ Не начинать работу без чёткого ответа
```

**Чеклист завершения:**
```
☐ Ответ дан на вопрос заказчика
☐ Решение зафиксировано в комментарии
☐ Карточка перемещена в «Закрыто»
☐ Заказчик уведомлён
```

После создания — **пометить карточку как шаблон** (Card Template).

**Название карточки:** `[ШАБЛОН] Не моё — {Тип задачи}`

**Метка:** Не моё (красная)

**Описание карточки:**
```

- **Тип:** Не моё (вне зоны ответственности AXIS)

⚠ Указать точный адресат
```

**Чеклист завершения:**
```

☐ Указан точный адресат (кто решит проблему)
☐ Адресат записан в карточке
☐ Карточка перемещена в «Возвращено заказчику»
```

## Задача 5: Настроить автоматизации (Butler)

Если доступен Trello Butler, настрой следующие правила:

### Автоматизация 1: Напоминание о необработанной задаче
```
Триггер: Карточка в «Новое» > 24 часов
Действие: добавить комментарий "@ruslan_archviz — ⚠ Есть необработанная входящая задача"
```

### Автоматизация 2: Напоминание о просроченном дедлайне
```
Триггер: Дедлайн карточки наступил
Действие: добавить красную метку "ПРОСРОЧЕНО"
```

### Автоматизация 3: Уведомление при переносе в «Возвращено заказчику»
```
Триггер: Карточка перемещена в «Возвращено заказчику»
Действие: добавить комментарий "⬅️ Задача возвращена заказчику: {кому}"
```

### Автоматизация 4: Комментарий при закрытии консультации
```
Триггер: Карточка перемещена в «Закрыто»
Действие: добавить комментарий "✅ Консультация завершена: {дата}"
```
