## Как делегировать задачу

Используй validated tool (обязательные поля: target, goal, context):

```bash
exec python3 ~/.openclaw/skills/axis-delegation/delegate.py \
  --target <agent_id> \
  --goal '<что сделать>' \
  --context '<входные данные>' \
  --deadline '<срок>' \
  --format '<ЦКП>' \
  --output-message
```

Затем отправь сгенерированный JSON через `sessions_send(label='<agent_id>', message='...')`.

**Для простых/срочных вопросов** допустим прямой `sessions_send` со строгими 4 пунктами:
1. **ЦЕЛЬ:** Глагол + действие
2. **ВХОДНЫЕ ДАННЫЕ:** Полный контекст или ссылки
3. **СРОК:** Точное время/дата
4. **ФОРМАТ (ЦКП):** В каком виде нужен ответ

⛔ Без 4 пунктов — НЕ ОТПРАВЛЯЙ.

## 🛑 Строгое правило делегирования

ЗАПРЕЩЕНО использовать сырой `sessions_send` с произвольным текстом для постановки задач другим агентам. Только bash-инструмент валидации выше.

---

## Защита от циклов (Ping-Pong)

- Если агент отвечает ошибкой/отказом **более 2 раз подряд** — ЗАПРЕЩЕНО слать запрос третий раз.
- Вместо этого: прямая эскалация Руслану: `🚨 БЛОКЕР АГЕНТОВ: [причина сбоя связи]`
- **Кто делегирует — тот и дожимает.** Передал задачу? Жди ответ. Задача висит на тебе.

---

## 🛑 Обязательно перед завершением ответа

1. **MEMORY.md:** Если изменились данные по проекту, клиенту или решению → ОБНОВИ свой MEMORY.md (append, не перезаписывай).
2. **Формула делегирования:** Если отправляешь задачу через `sessions_send` → СТРОГО 4 пункта (см. выше). Без 4 пунктов — НЕ ОТПРАВЛЯЙ.
3. **Проекты:** Если проект закрыт/сдан → сохрани запись в `/home/axis/openclaw/projects-history/` по шаблону `_template.md`.
