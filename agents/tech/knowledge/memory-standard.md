## 📋 ОБЯЗАТЕЛЬНАЯ СТРУКТУРА

Каждый агент ОБЯЗАН иметь в MEMORY.md следующие 4 секции:

```markdown
## Последнее обновление: YYYY-MM-DD

## Текущие задачи
- [ ] Задача 1 — срок, статус
- [ ] Задача 2 — срок, статус

## Ключевые решения за неделю
- [дата] Решение 1 — контекст, результат
- [дата] Решение 2 — контекст, результат

## Открытые вопросы к Руслану
- Вопрос 1 (если нет — написать "Нет открытых вопросов")
```

---

## 📏 ПРАВИЛА

### Обязательные
1. **4 секции всегда присутствуют** — даже если пустые (писать "Нет" / "—")
2. **Дата обновления** — реальная дата последнего содержательного изменения
3. **Текущие задачи** — только активные (закрытые переносить в архив `memory/`)
4. **Ключевые решения** — факты с датой, не формальные отписки
5. **Вопросы к Руслану** — конкретные, с контекстом

### Запрещено
- ❌ Формальные записи типа "Ежедневный аудит — штатный режим" без содержания
- ❌ Копипаст одного и того же текста каждый день
- ❌ Пустой MEMORY.md (0 строк)
- ❌ Устаревшие задачи (закрытые >7 дней назад в "Текущих")

### Рекомендуемый объём
| Метрика | Минимум | Норма | Максимум |
|---------|---------|-------|----------|
| Строк | 15 | 30-100 | 300 |
| Текущих задач | 1 | 3-7 | 15 |
| Решений за неделю | 1 | 3-5 | 10 |

---

## 📁 АРХИВАЦИЯ

Когда MEMORY.md превышает 200 строк:
1. Перенести закрытые задачи и старые записи в `memory/YYYY-MM-DD.md`
2. Оставить в MEMORY.md только актуальное
3. В начале MEMORY.md указать: `Архив: memory/`

## 🔍 ИНСТРУКЦИЯ ДЛЯ QC

### Что проверять (ежедневный аудит 20:00):

**Шаг 1 — Структура:**
- [ ] Есть секция "Последнее обновление" с датой
- [ ] Есть секция "Текущие задачи"
- [ ] Есть секция "Ключевые решения за неделю"
- [ ] Есть секция "Открытые вопросы к Руслану"

**Шаг 2 — Содержание:**
- [ ] Дата обновления не старше 48 часов (для активных агентов)
- [ ] Текущие задачи содержат конкретику (не "работаю над проектом")
- [ ] Решения за неделю — реальные факты с датами
- [ ] Нет копипаста предыдущего дня

**Шаг 3 — Объём:**
- [ ] Минимум 15 строк
- [ ] Не более 300 строк (если больше — напомнить про архивацию)

### Классификация нарушений:

| Нарушение | Уровень | Действие |
|-----------|---------|----------|
| MEMORY.md пустой / нет файла | 🔴 Критическое | Немедленно уведомить агента + Tech |
| Нет 1+ обязательной секции | 🟡 Серьёзное | Уведомить агента, срок исправления 24ч |
| Формальная запись без содержания | 🟡 Серьёзное | Уведомить агента, потребовать реальные данные |
| Дата обновления >48ч | 🟡 Серьёзное | Уведомить агента |
| Объём >300 строк | 🔵 Рекомендация | Предложить архивацию |

### Формат отчёта QC:
```
📋 MEMORY AUDIT — [дата]
✅ Норма: [список агентов]
🟡 Нарушения: [агент] — [причина]
🔴 Критичное: [агент] — [причина]
```
