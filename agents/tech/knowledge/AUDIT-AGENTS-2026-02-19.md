- ✅ Делегирование задач агентам
- ✅ Модерация планёрок

**Почему это перегруз:**
- 3 ритуала управления в неделю (ежедневный, еженедельный, ежемесячный)
- Постоянная коммуникация с 4 агентами (Tech, QC, HR, Finance)

**Что делает агент "лишнего":**

## 📡 АНАЛИЗ КОММУНИКАЦИИ

### Текущая коммуникационная сеть

```
Main → Ops → [Tech, QC, HR, Finance]
               ↖───────────────┘
Sales → Finance
QC → Tech (эскалация)
HR → Ops (нагрузка)
```

### Накладные расходы на коммуникацию

**Сейчас (Ops-центричная архитектура):**
- Любая задача от команды → Ops
- Ops делегирует агенту
- Оператор следит за выполнением
- Агент возвращает результат Ops
- Ops передаёт в Main

**Плюсы:**
- Единая точка входа
- Контроль соблюдения регламентов

**Минусы:**
- Когнитивная нагрузка на Ops (должен понимать каждую задачу)
- Задержка (Ops → Агент → Ops → Main)
- Узкое горлышко (Ops перегружен)

**После декомпозиции:**

```
Main → Sales → Finance
Main → HR → Team
Main → QC → Tech
Main → Ops (ритуалы)
```

**Плюсы:**
- Быстрая коммуникация (Sales → Finance напрямую)
- Оперативность
- Разгрузка Ops

**Минусы:**
- Сложнее отслеживать поток задач (каждый агент автономен)
- Риск рассинхронизации (если два агента делают одно и то же)

**Вывод:**
- Для Sales/Finance/HR → прямая коммуникация ОПТИМАЛЬНА
- Для QC → эскалация Tech
- Для Ops → нужен координатор, но не тот который делает ВСЁ

## ✅ КРИТЕРИИ МАСШТАБИРОВАНИЯ (ЧЕК-ЛИСТ)

Используй этот чек-лист перед созданием нового агента:

### 1. ОБЪЁМ РАБОТЫ
- [ ] Среднее количество задач в неделю **≥15** (иначе один скилл)
- [ ] Общее время на задачу **≥30 минут** (иначе один скилл)
- [ ] Задачи возникают **≥3 раза в неделю**

### 2. ТИП РАБОТЫ
- [ ] Разные типы задач (творческие + аналитические)
- [ ] Требуется разный стиль общения (эмоциональный + формальный)
- [ ] Требуется разный инструментарий

### 3. КОНФЛИКТНОСТЬ ЗОН ОТВЕТСТВЕННОСТИ
- [ ] Разные подзадачи одного процесса
- [ ] Задачи, требующие разных компетенций
- [ ] Зоны ответственности пересекаются >20% времени

### 4. КОНФИДЕНЦИАЛЬНОСТЬ
- [ ] Требуется строгая конфиденциальность
- [ ] Необходимо дополнительное управление доступом

### 5. ИНСТРУМЕНТЫ
- [ ] Необходим уникальный инструмент/интеграция
- [ ] Требуется обучение пользователей

### РЕЗУЛЬТАТ:
- Если **4+** пунктов — **СОЗДАЙ НОВОГО АГЕНТА**
- Если **2-3** — **ОБЪЕДИНИ СКИЛЛЫ**
