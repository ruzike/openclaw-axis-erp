> 📌 **Главное правило**
> AI-агент (@axis_opps_bot) — это твой рабочий инструмент. Он получает задачи, помнит дедлайны, напоминает и отчитывается. Чем точнее ты пишешь — тем лучше он работает.

## 1. Что такое AI-агент и зачем он нужен

AI-агент — это бот в Telegram (@axis_opps_bot), который работает круглосуточно и помогает студии работать без постоянного участия руководителя. Он не просто отвечает на вопросы — он управляет задачами, напоминает о дедлайнах и фиксирует нарушения.

| Возможность | Описание |
|---|---|
| Принимать задачи | Ты пишешь задачу — агент создаёт карточку в Trello с дедлайном и ответственным |
| Напоминать | Утром напоминает о задачах на день, вечером просит отчёт о выполнении |
| Отвечать на вопросы | Знает регламенты студии, может объяснить процессы C0–C5 |
| Фиксировать статусы | Записывает кто и что выполнил, передаёт отчёты руководителю |
| Эскалировать | Если задача просрочена — автоматически уведомляет руководителя |
| Контроль качества | QC Officer проверяет выполнение стандартов и фиксирует отклонения |

## 2. Система AI-агентов AXIS

Ты взаимодействуешь не с одним ботом, а с системой из 8 специализированных агентов. Каждый отвечает за свою зону:

| Агент | Бот | Роль | ЦКП (результат) |
|---|---|---|---|
| **Main** | @axis_main_bot | Координация всей системы | Работающая система бизнеса |
| **Ops** | @axis_opps_bot | Ритм компании, задачи | Выполненные в срок задачи |
| **QC** | @axis_qc_bot | Контроль качества | 0 критических ошибок |
| **Tech** | @axis_arch_tech_bot | Регламенты, автоматизация | Актуальная база знаний |
| **HR** | @axis_teams_bot | Найм, адаптация | Укомплектованный штат |
| **Finance** | @axis_finance_bot | Учёт, бюджеты | Точные финансовые отчёты |
| **Sales** | @axis_sales_bot | Продажи, клиенты | Выполненный план по выручке |
| **Team** | — | Личный ассистент собственника | Свободное время собственника |

> 💡 **Основной контакт для сотрудников — Ops Agent** (@axis_opps_bot). Он направит тебя к нужному агенту, если вопрос не в его зоне.

## 3. Как начать работу

1. Открой Telegram и найди @axis_opps_bot
2. Нажми /start
3. Напиши своё имя: «Привет, я [имя], начинаю работу»
4. Агент подтвердит подключение и объяснит что делать дальше

**✅ Пример первого диалога:**

```
Ты:    /start
Агент: Привет! Я операционный координатор студии AXIS. Напиши своё имя.
Ты:    Привет, я Мирас
Агент: Отлично, Мирас! Ты подключён. Задачи на сегодня: [список задач из Trello]
```

## 4. PDCA-цикл — основа работы

Каждая задача в AXIS проходит через цикл **PDCA**:

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  PLAN   │ ──► │   DO    │ ──► │  CHECK  │ ──► │   ACT   │
│Планируй │     │Выполняй │     │Проверяй │     │Улучшай  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
                                                      │
                                                      ▼
                                              Новый цикл PDCA
```

| Этап | Что делаешь ты | Что делает агент |
|---|---|---|
| **Plan** | Формулируешь задачу по формуле делегирования | Создаёт карточку в Trello, фиксирует ЦКП и дедлайн |
| **Do** | Выполняешь задачу, фиксируешь сложности | Напоминает, контролирует сроки |
| **Check** | Отчитываешься о результате | QC Officer проверяет качество, сравнивает план/факт |
| **Act** | Делаешь выводы, предлагаешь улучшения | Фиксирует выводы, обновляет стандарты |

## 5. Еженедельный цикл (ритуалы управления)

> **⚡ ЗОЛОТОЕ ПРАВИЛО:** Ритуалы управления — приоритет #1. Ничто не может их отменить.

### Понедельник 10:00 — Еженедельная планёрка (60-90 мин)

**Протокол:**
1. **Отчёт по метрикам** (15 мин) — ключевые KPI, что выросло/упало
2. **Отчёт по задачам прошлой недели** (30 мин) — что сделано, что НЕ сделано и почему
3. **Планы на неделю** (30 мин) — новые задачи по формуле делегирования

**Выход:** Задачи на неделю в Task Manager.

### Ежедневно 09:00 — Брифинг (15 мин)

Каждый по кругу (1-2 мин):
1. Что сделал вчера (факты)
2. Что планирую сегодня (задачи)
3. Есть ли блокеры

### Пятница 17:00 — Мини-ретро (Check + Act)

1. Что получилось на этой неделе?
2. Что не получилось и почему?
3. Что улучшить на следующей неделе?
4. Предложения по изменению процессов/регламентов

### Ежемесячно (1-е число) — Большая ретроспектива (2-3 часа)

1. Отчёт по цифрам за месяц
2. Отчёт по задачам месяца
3. Планы на месяц по задачам развития
4. Обновление ключевых метрик

## 6. Ежедневный ритуал — обязательно

Каждый рабочий день происходит по одной схеме. Агент инициирует сам, тебе нужно только отвечать.

### Утро (09:00) — Брифинг

```
Агент: Доброе утро, Мирас! Задачи на сегодня:
       1. Визуализация ЖК Токио — дедлайн 18:00
       2. Ответить клиенту по SE Tower
       Что берёшь в работу?

Ты:    Беру обе задачи. Начинаю с визуализации.
```

### Вечер (18:00) — Отчёт

```
Агент: Мирас, вечерний отчёт. Как прошёл день?
       1. Визуализация ЖК Токио — выполнено?

Ты (выполнено):    Да, визуализация готова. Отправил клиенту.
Ты (не успел):     Не успел, перенос на завтра. Причина: клиент прислал правки в 17:00.
```

> ⚠️ **Важно**
> - Не ответил на утренний брифинг до 10:00 → агент напомнит ещё раз
> - Не ответил на вечерний отчёт до 19:00 → руководитель получит уведомление
> - Это не штраф, это система контроля. Просто отвечай.

## 7. Формула делегирования — как ставить задачи

Каждая задача формулируется по **4 шагам**:

| Шаг | Вопрос | Пример |
|---|---|---|
| 1. ЗАЧЕМ | Какой смысл? Как влияет на цели? | Клиент ждёт рендеры для презентации инвестору |
| 2. ЧТО (ЦКП) | Что конкретно получим в конце? | 12 рендеров 4K в формате JPG |
| 3. Результативный глагол | Начинай с: создать, завершить, внедрить | **Подготовить** финальные рендеры |
| 4. Первый шаг | С чего начнёшь? (проверка понимания) | Начну с экстерьера, блок А |

### 5 правил постановки задач

1. Задача **ЗАПИСАНА** в Task Manager (не в голове!)
2. Задача **САМОСТОЯТЕЛЬНА** (понятна без контекста через неделю)
3. Задача по **ФОРМУЛЕ ДЕЛЕГИРОВАНИЯ**
4. Есть **СРОК** (дата + время)
5. Есть **ОДИН ОТВЕТСТВЕННЫЙ** (не "мы", не "отдел")

### Чек-лист правильной задачи

- [ ] Минимум 5 слов в названии
- [ ] Начинается с результативного глагола
- [ ] Перенесена в Task Manager
- [ ] Понятен результат (ЦКП)
- [ ] Понятна без контекста
- [ ] Точный срок (дата + время)
- [ ] Один ответственный (ФИО или Агент)

| ❌ Неправильно | ✅ Правильно |
|---|---|
| Сделай рендеры | Подготовить 12 рендеров ЖК Токио в 4K до пятницы 18:00 |
| Позвони клиенту | Дозвониться до клиента CED Tower, согласовать правки, занести в CRM до среды |
| Разберись с Trello | Обновить статусы всех карточек на доске Производство до конца дня |
| Нужен отчёт | Создать еженедельный отчёт по активным проектам до пятницы 17:00 |

## 8. KPI и метрики сотрудника

Каждый сотрудник оценивается по измеримым показателям:

### Базовые метрики (для всех)

| Метрика | Целевое значение | Как считается |
|---|---|---|
| Выполнение задач в срок | ≥ 85% | Завершённые вовремя / Все задачи за неделю |
| Ответ на брифинг | 100% | Ответил до 10:00 каждый день |
| Вечерний отчёт | 100% | Отправил до 19:00 каждый день |
| Просроченные задачи | ≤ 2 в неделю | Задачи с нарушением дедлайна |
| Качество (оценка QC) | ≥ 4 из 5 | По результатам проверок QC Officer |

### Как агент отслеживает KPI

- **Еженедельно (пятница)** — Ops Agent присылает сводку по твоим метрикам
- **Ежемесячно** — Main Agent формирует общий отчёт для руководителя
- **При отклонении** — QC Officer фиксирует нарушение и уведомляет

```
Агент: Мирас, твои метрики за неделю:
       ✅ Выполнение задач: 90% (9 из 10)
       ✅ Брифинги: 5/5
       ⚠️ Вечерние отчёты: 4/5 (пропуск — среда)
       📊 Оценка QC: 4.2/5
       Общий балл: Хорошо 👍
```

## 9. Контроль качества — QC Officer

**QC Officer** (@axis_qc_bot) — это «полицейский» системы. Он проверяет:

| Что проверяет | Как проверяет | Последствия |
|---|---|---|
| Качество выполненных задач | Выборочный аудит результатов | Оценка 1-5 в карточке задачи |
| Соблюдение регламентов | Проверка переписок, процессов | Фиксация в Журнал Косяков |
| Сроки выполнения | Данные из Task Manager | Уведомление руководителю |
| Формулировки задач | Проверка по чек-листу делегирования | Возврат задачи на переформулировку |

### Как это выглядит для тебя

```
QC Officer: Мирас, проверил рендеры ЖК Токио.
            ✅ Качество: 4/5 — хорошо
            ⚠️ Замечание: нет ночной визуализации (было в ТЗ)
            📋 Задача на доработку создана, дедлайн: завтра 15:00

Ты:         Принял, сделаю.
```

> 💡 **QC — это не наказание, а помощь.** Цель — поймать ошибку до клиента.

## 10. Примеры рабочих диалогов

### Сценарий 1 — Поставить задачу (по формуле делегирования)

```
Ты:    Создай задачу: согласовать планировку с клиентом CED Tower до четверга.
       Цель: клиент утверждает планировку перед началом 3D-работ.

Агент: Создал карточку в Trello:
       📋 «Согласовать планировку CED Tower с клиентом»
       🎯 ЦКП: подписанный PDF «Согласовано»
       📅 Дедлайн: четверг, 20 февраля
       👤 Ответственный: ты
       Подтверждаешь?

Ты:    Да. Первый шаг — отправлю клиенту планировку на проверку сегодня.

Агент: Готово! Напомню тебе в среду в 09:00.
```

### Сценарий 2 — Узнать статус задач

```
Ты:    Что у меня на сегодня?

Агент: Твои задачи на сегодня:
       1. Рендеры ЖК Токио — дедлайн 18:00 🔴 (срочно)
       2. Ответ клиенту SE Tower — дедлайн сегодня
       3. Обновить Trello — без дедлайна
```

### Сценарий 3 — Перенести задачу

```
Ты:    Перенеси рендеры Токио на завтра, не успеваю.

Агент: Перенёс. Новый дедлайн: завтра 18:00.
       Причина переноса? (нужно для отчёта руководителю)

Ты:    Клиент прислал дополнительные материалы поздно.

Агент: Зафиксировал. Руководитель будет уведомлён.
```

### Сценарий 4 — Вопрос по регламенту

```

Агент: C0 — это процесс приёма и маршрутизации задачи. Краткий алгоритм:
       1. Получить бриф от клиента
       2. Проверить по чек-листу (все ли данные есть)
       3. Создать карточку в Trello
       4. Назначить ответственного
       5. Подтвердить клиенту срок
       Подробный регламент: [ссылка на Notion]
```

## 11. Чего НЕ нужно делать

> 🚫 **Типичные ошибки**
> - Игнорировать утренний брифинг — система отметит как нарушение
> - Отвечать «ок» без содержания — агент не поймёт что конкретно выполнено
> - Ставить задачи без дедлайна — всегда указывай срок
> - Писать задачи без результативного глагола — «позвонить» ≠ «дозвониться и согласовать»
> - Писать только в Telegram напрямую, минуя бота — задачи не попадут в Trello
> - Молчать при проблемах — лучше написать «задача блокирована, причина Х»
> - Игнорировать замечания QC Officer — они фиксируются в Журнал Косяков

## 13. Механизм улучшения регламента

Этот регламент — живой документ. Он обновляется по циклу PDCA:

1. **Предложи улучшение** → напиши агенту: «Предлагаю изменить регламент: [описание]»
2. **Tech Agent** рассмотрит предложение и оформит драфт
3. **QC Officer** проверит на непротиворечивость
4. **Main Agent** утвердит изменение
5. **Ops Agent** уведомит команду о новой версии

> 💡 Каждую пятницу на мини-ретро можно озвучить предложения по улучшению процессов. Лучшие идеи внедряются в следующем PDCA-цикле.

**Версионирование:**
- Все версии хранятся в Notion
- Изменения отмечаются в changelog
- Сотрудники уведомляются о каждом обновлении

## 14. Если что-то не работает

1. Попробуй написать /start — это перезапускает сессию
2. Если бот не отвечает больше 5 минут — напиши напрямую руководителю
3. Если задача не создалась в Trello — скажи агенту: «повтори создание задачи»
4. Если не знаешь как сформулировать задачу — спроси агента: «помоги сформулировать задачу»
