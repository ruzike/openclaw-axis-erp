# Регламент: Грумминг бэклога (по методологии АСБ)

**Периодичность:** Еженедельно
**День:** Пятница
**Время:** 16:00
**Длительность:** 30 минут
**Участники:** Руслан, Ops Координатор
**Платформа:** Trello

---

## Цель

Поддержание бэклога в актуальном состоянии. Бэклог = единое информационное пространство со ВСЕМИ задачами от А до Я.

## Структура бэклога (по АСБ)

- Рабочие задачи (рутинные)
- Задачи развития
- Проекты (с декомпозицией)
- Задачи по обучению

## Kanban-доска (движение задач)

```
Бэклог → На месяц → На неделю → На сегодня → Выполнено
```

**Ритуалы двигают задачи:**
- Ретроспектива: Бэклог → На месяц
- Планёрка: На месяц → На неделю
- Брифинг: На неделю → На сегодня
- После выполнения → Выполнено

## Структура грумминга

### 1. Обзор бэклога (10 мин)
- Просмотр всех карточек в "Бэклог"
- Отметка устаревших

### 2. Приоритизация (10 мин)
- Ранжирование по важности/срочности
- Стыковка с целями компании
- Перемещение приоритетных → "На месяц" / "На неделю"

### 3. Очистка (5 мин)
- Неактуальные → архив
- Дубликаты → объединение

### 4. Новые задачи (5 мин)
- Добавление из накопленных идей
- Формулировка по формуле делегирования

## Формула делегирования (чек-лист задачи)

- [ ] Минимум 5 слов
- [ ] Начинается с результативного глагола
- [ ] Понятно, какой должен быть результат
- [ ] Можно вырвать из контекста — всё понятно
- [ ] Есть срок
- [ ] Есть ответственный
