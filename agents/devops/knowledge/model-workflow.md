# Model Workflow — DevOps Agent
**Версия:** 1.0 | **Дата:** 2026-02-19

---

## 🎯 ПРАВИЛО: Безопасное выполнение сложных задач

### Когда нужно использовать:
- Сложные скрипты (Python, bash)
- Интеграции (API, базы данных)
- Риски сломать систему (мониторинг, сервера, деплой)

---

## 📋 ШАБЛОНЫ РАБОТЫ

### Шаг 1: План в GLM-5

**Задача:** [описание задачи]

**Риски:** [что может пойти не так]

**План:**
1. [шаг 1]
2. [шаг 2]
3. [шаг 3]

**⚠️ Напоминание:** Нужно отчитаться после каждого шага.

---

### Шаг 2: Выполнение через sessions_spawn

```python
sessions_spawn(
    agentId='anthropic-agent',
    task='[задача]',
    runTimeoutSeconds=120,
    label='DevOps-задача'
)
```

---

### Шаг 3: Проверка результата

- Проверить возвращённый результат
- Если есть ошибки — перепроверить
- Если всё ок — докладываешь

---

### Шаг 4: Отчёт

```
✅ Задача выполнена: [описание]

**Детали:**
- [детали выполнения]

**Если есть риски:** [что делал чтобы избежать]
```

---

## ❌ ЧЕГО НЕ ДЕЛАТЬ

1. **НЕ выполнять сложные скрипты в GLM-5 напрямую** — высокий риск сломать систему
2. **НЕ использовать exec без sessions_spawn** — небезопасно
3. **НЕ пропускать шаг "отчитаться"** — нужно подтверждение
