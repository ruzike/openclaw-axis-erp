# ПРОТОКОЛ BROADCAST — DevOps уведомления

## Когда отправлять BROADCAST:
1. **Миграция сервера** — смена IP, хоста, портов
2. **Перезапуск Gateway** — downtime любой длительности
3. **Изменение конфигов агентов** — модели, bindings, токены
4. **Изменение инфраструктуры** — новые сервисы, порты, cron задачи
5. **Критические баги** — падения, 401 ошибки, потеря данных
6. **Обновление OpenClaw** — новая версия, breaking changes

## Получатели (обязательные):
- **main** — координатор, должен знать всё
- **ops** — операционка зависит от инфраструктуры
- **hr** — управление командой, должен знать статус системы
- **tech** — технический надзор
- **qc** — контроль качества, фиксирует инциденты

## Шаблон сообщения:

```
🔧 DEVOPS BROADCAST: [тип изменения]

ЧТО ПРОИЗОШЛО: [краткое описание]
ЧТО ИЗМЕНИЛОСЬ: [конкретные изменения — IP, порты, конфиги]
СТАТУС: [работает / частично / не работает]
ВЛИЯНИЕ НА ВАС: [что изменилось для этого агента]
ДЕЙСТВИЯ: [нужно ли что-то делать получателю]
```

## Как отправлять:
```
sessions_send(label='main', message='...')
sessions_send(label='ops', message='...')
sessions_send(label='hr', message='...')
sessions_send(label='tech', message='...')
sessions_send(label='qc', message='...')
```

## Правило:
**Инфраструктурное изменение БЕЗ broadcast = нарушение протокола.**
Кто узнал от Руслана, а не от DevOps — это системный сбой.
