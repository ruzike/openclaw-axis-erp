## 1. ФИЛОСОФИЯ АСБ: АВТОНОМНАЯ КОМПАНИЯ

### Главная цель
**95% автономность компании** — собственник занимается только стратегией, всё остальное работает без него.

### Признаки внедрённого проектного управления

**✅ Когда всё работает:**
- Все задачи записаны в таск-менеджере
- Приоритеты расставлены между задачами
- Руководитель в курсе, кто и чем занимается
- Проблемы обсуждаются и решаются
- У каждой задачи есть ответственный и срок
- Все задачи сформулированы по формуле делегирования
- Ритуалы управления реально работают (брифинги, планёрки, совещания)

### Элементы проектного управления (все обязательны!)

1. **Task Manager** — система хранения задач
2. **Формула делегирования** — как формулировать задачи
3. **Правила постановки и контроля** — протоколы
4. **Брифинги и планёрки** — ежедневные и еженедельные
5. **Совещания** — периодические координации
6. **Ретроспективы** — ежемесячный анализ
7. **Регламенты** — правила для всех ритуалов

> **КРИТИЧНО:** Все элементы нужно внедрять **КОМПЛЕКСНО**. Таск-менеджер без ритуалов НЕ ЗАРАБОТАЕТ!

### Почему таск-менеджеры не работают

**❌ Типичная ошибка:**
- Внедрили таск-менеджер
- Запихнули туда задачи
- Люди не пользуются
- "Таск-менеджеры не работают!"

**✅ Правильно:**
- Внедрить ВСЕ ритуалы управления
- Назначить ответственного за проектное управление
- Внедрить ОКК на соблюдение правил
- ТОГДА заработает!

## 2. 8 АГЕНТОВ АСБ: РОЛИ И ФУНКЦИИ

### Система из 8 агентов для автономности

| # | Агент | Роль | Главная цель |
|---|-------|------|--------------|
| 1 | **Main/COO** | Операционный директор | Координация всей системы, делегирование |
| 2 | **Личный Ассистент** | Executive Assistant | Освободить 80% времени собственника |
| 3 | **Операционный Координатор** | Ops Manager | Ритуалы управления, команда, задачи |
| 4 | **ОКК Контролёр** | Quality Control Officer | Контроль качества, фиксация косяков |
| 5 | **Внедренец-Системщик** | Tech/Process | Регламенты, процессы, автоматизация |
| 6 | **HR & Рекрутер** | HR Manager | Найм, адаптация, развитие команды |
| 7 | **Финансовый Аналитик** | CFO | Цифры, бюджеты, финансовая модель |
| 8 | **Продажный Эксперт** | Sales Manager | Лиды, продажи, конверсия |

### Взаимодействие агентов

```
Собственник
    ↓
Личный Ассистент (фильтр)
    ↓
Main/COO (координация)
    ├→ Операционный Координатор (команда)
    │   ├→ Внедренец (процессы)
    │   ├→ HR (люди)
    │   └→ ОКК (качество)
    ├→ Финансовый Аналитик (цифры)
    └→ Продажный Эксперт (продажи)
```

### Приоритет координации

> **ЗОЛОТОЕ ПРАВИЛО:** Ритуалы управления — это приоритет #1 компании!

Если планёрка в четверг в 15:00 — ничто не может её отменить:
- Не клиент
- Не поставщик
- Не форс-мажор (кроме критичного)

Пока координация не станет приоритетом #1 — система не заработает!

## 3. ФОРМУЛА ДЕЛЕГИРОВАНИЯ

### Определение

**Формула делегирования** — алгоритм формулировки задачи, который кратно повышает шансы её эффективной реализации.

**Основная причина невыполнения** = неправильная формулировка!

### 4 шага формулы

**1. ЗАЧЕМ нужна эта задача?**
- Как повлияет на статистики?
- Если нет ответа — задачу НЕ брать!

**2. ЧТО ожидаем в результате?**
- Оцифрованный, измеримый результат
- Конкретное описание outcome

**3. Записать с РЕЗУЛЬТАТИВНОГО глагола**
- ✅ Сделать, построить, внедрить, создать, организовать
- ❌ НЕ процессный: уведомлять, позвонить, обсудить

**4. Спросить ПЕРВЫЙ ШАГ у исполнителя**
- "С чего начнёшь?"
- Проверка понимания задачи

### 5 правил постановки задач

1. Задача должна быть **ЗАПИСАНА** (не в голове!)
2. Задача должна быть **САМОСТОЯТЕЛЬНА** (понятна вне контекста)
3. Задача записана **ПО ФОРМУЛЕ ДЕЛЕГИРОВАНИЯ**
4. У задачи есть **СРОК ИСПОЛНЕНИЯ**
5. У задачи есть **ОТВЕТСТВЕННЫЙ**

### Итоговый чек-лист задачи

- [ ] Минимум 5 слов
- [ ] Начинается с результативного глагола
- [ ] Перенесена в таск-менеджер
- [ ] Понятно, какой должен быть результат
- [ ] Можно вырвать из контекста — всё понятно
- [ ] Есть срок
- [ ] Есть ответственный

### Ошибки (задачи, которые НЕ будут выполнены)

**❌ Плохие примеры:**
- "Подготовить отчёт" (2 слова, неясный результат)
- "Позвонить клиенту" (процессный глагол, нет результата)
- "Уведомлять отдел кадров о командировках" (процессный, бесконечный)
- В личном блокноте (не в таск-менеджере)
- Без срока и ответственного
- Проект вместо задачи (нужна декомпозиция!)

**✅ Хорошие примеры:**
- "Создать отчёт по продажам за январь 2026 с разбивкой по менеджерам и отправить собственнику до 5 февраля"
- "Организовать оперативное (не менее чем за 3 рабочих дня) уведомление отдела кадров об отправлении сотрудников в командировки через единый календарь"
- "Внедрить систему автоматической отправки напоминаний клиентам за 3 дня до оплаты счёта"

### Различие: процессный vs результатный глагол

| Процессный (❌) | Результатный (✅) |
|----------------|------------------|
| Уведомлять | Организовать уведомление |
| Позвонить и обсудить | Отправить заявку и добиться выполнения |
| Проверять отчёты | Внедрить систему контроля отчётов |
| Следить за проектом | Завершить проект к дедлайну |

## 4. PDCA-ЦИКЛ

### Определение

**PDCA** = Plan → Do → Check → Act

Цикл непрерывного улучшения для любой задачи, процесса, компании.

### 4 этапа

**1. PLAN (Планирование):**
- Цель → причины проблемы
- Базовые KPI (текущее состояние)
- Что нужно изменить?
- План изменений

**2. DO (Выполнение):**
- Выполнение запланированных мероприятий
- Фиксация того, что делаем

**3. CHECK (Проверка):**
- Сбор данных
- Анализ результатов
- Гипотеза подтверждена?
- KPI достигнуты?

**4. ACT (Действие):**
- Выводы из цикла
- Корректировка плана
- Следующий цикл PDCA

### Принцип: "А не ерунду ли я делаю?"

> Вовремя остановиться и перестать делать то, что не приведёт к результату — мега полезная штуковина!

### Применение

- При управлении компанией (в целом)
- При управлении одним процессом
- При управлении ЛЮБОЙ задачей
- Каждая задача проходит через PDCA

**Пример:**
```
PLAN: Хотим увеличить конверсию лид→сделка с 20% до 30%
      Причина: медленный ответ на лиды (>24 часа)
      Решение: внедрить автоответ в течение 1 часа
      
DO:   Настроили CRM, обучили менеджеров, запустили

CHECK: Через месяц конверсия = 28%
       Время ответа <1 часа = 95% лидов
       Гипотеза частично подтверждена
       
ACT:  Выводы: автоответ работает, но недостаточно
      Корректировка: добавить персонализацию ответа
      Следующий PDCA: тестируем персонализацию
```

## 5. БЭКЛОГ И KANBAN

### Бэклог (Backlog)

**Определение:** Единое информационное пространство, где содержатся ВСЕ задачи от А до Я.

**Структура бэклога:**
- Рабочие задачи (рутинные)
- Задачи развития
- Личные задачи (не показывать сотрудникам!)
- Задачи по обучению
- Проекты (с декомпозицией)

**Грумминг бэклога** (в рамках ритуалов):
- Брать самые приоритетные задачи
- Проверять актуальность
- Стыкуются ли с целями компании?
- Удалять неактуальные задачи
- Сразу разносить по сферам и типам

### Kanban-доска (структура)

| Столбец | Описание | Когда переносим |
|---------|----------|----------------|
| **Бэклог** | Все задачи компании | - |
| **На месяц** | Утверждённые на месячном планировании | Ежемесячная ретроспектива |
| **На неделю** | Перенесённые на недельной планёрке | Еженедельная планёрка |
| **На сегодня** | Дневной фокус | Ежедневный брифинг |
| **Выполнено** | Завершённые задачи | После выполнения |

### Движение задач по доске

**Ритуалы управления двигают задачи:**

```
Бэклог 
  ↓ (месячная ретроспектива)
На месяц
  ↓ (недельная планёрка)
На неделю
  ↓ (ежедневный брифинг)
На сегодня
  ↓ (после выполнения)
Выполнено
```

**БЕЗ РИТУАЛОВ ЗАДАЧИ НЕ ДВИГАЮТСЯ!**

### Обязательные атрибуты задачи в таск-менеджере

- Ответственный (кто делает)
- Дедлайн (когда закончить)
- Метки (продажи, маркетинг, производство, и т.д.)
- Чек-листы (если сложная задача)
- Формулировка по формуле делегирования

## 6. РИТУАЛЫ УПРАВЛЕНИЯ

### Таблица ритуалов управления

**КРИТИЧНО:** Все ритуалы должны быть выгружены в таблицу!

**Структура таблицы:**

| Отдел | Участники | Ритуал | Периодичность | Дата/День | Вход | Выход | Регламент |
|-------|-----------|--------|---------------|-----------|------|-------|-----------|
| Компания | Руководители | Недельная планёрка | 1/неделю | Пн 09:00 | Отчёты | Протокол + задачи | [ссылка] |
| Отдел продаж | Менеджеры | Ежедневный брифинг | Ежедневно | 09:00 | - | Задачи на день | [ссылка] |

### Виды ритуалов

1. **Ежедневные координации (Брифинги)** — 15 минут
2. **Еженедельные координации (Планёрки)** — 60-90 минут
3. **Ежемесячные собрания (Ретроспективы)** — 2-3 часа
4. **Периодические совещания** — по необходимости
5. **Периодическая отчётность** — автоматизировать

### 6.1. НЕДЕЛЬНОЕ ПЛАНИРОВАНИЕ (ключевой ритуал!)

**Цель:** Скоординировать все задачи и оперативно реагировать на изменения в статистиках.

**Когда:** В одно и то же время каждую неделю  
**Формат:** Онлайн (с камерами), все сфокусированы  
**Длительность:** 60-90 минут (строго!)  

**Правила:**
- Начинается **строго вовремя** (не ждём опоздавших)
- Говорит **один человек** (модератор пресекает балаган)
- Все участвуют в **корректной формулировке задач**
- Уложиться в **оговоренное время**

**Последовательность отчёта:**

**1. Отчёт по цифровым показателям (статистики) — 15 минут**
- Ключевые метрики компании
- Что выросло, что упало
- Причины отклонений

**2. Отчёт по выполненным задачам прошлой недели — 30 минут**
- Что выполнили
- Что НЕ выполнили (и почему!)
- Логически вытекающие задачи

**3. Планы на следующую неделю — 30 минут**
- Задачи по формуле делегирования
- Ожидаемый результат
- Приоритеты
- Возможные сложности

**Выход:** Поставленные сформулированные задачи на неделю с понятным, измеримым результатом

**После планёрки:**
- Протокол с задачами → Google Drive
- Все задачи → таск-менеджер
- Рассылка протокола участникам

---

### Шаги внедрения недельного планирования

1. **Разработать форму план-отчётов** (см. раздел 7)
2. **Составить драфт регламента планёрки** (подготовка, тайминг, выход)
3. **Составить регламент ОКК** на подготовку к планёрке
4. **Создать чаты в Telegram** для отчётов
5. **Внедрить ОКК** на соблюдение регламента

### Типичные проблемы на планёрках (и как решать)

**❌ Проблема: Балаган и обвинения**
- Решение: Модератор жёстко пресекает, возвращает к фактам

**❌ Проблема: Фонтанирование идеями вместо задач**
- Решение: "Отлично! Запиши в бэклог. Сейчас фокус на этой неделе"

**❌ Проблема: Нежелание говорить конкретно ("водичка")**
- Решение: "Сформулируй задачу по формуле делегирования прямо сейчас"

**❌ Проблема: Проекты вселенского масштаба вместо задач**
- Решение: "Декомпозируй на задачи. Что сделаем на этой неделе?"

---

### 6.2. БРИФИНГИ (ежедневные, 15 минут)

**Цель:** Синхронизация команды, быстрая координация

**Правила:**
- Каждый день в **одно время** (например, 09:00)
- Задачи по **формуле делегирования**
- Только **правильная и достоверная информация**
- Отвечать чётко: **"Сделал", "Планирую"**
- **Не вовлекать** коллег в обсуждения
- **Актуализировать** таск-менеджер после брифинга
- Обратная связь в **корректной форме**

**Формат:**
```
Каждый по кругу:
1. Что сделал вчера (факты)
2. Что планирую сегодня (задачи)
3. Есть ли блокеры (если да — обсудим после)

Длительность: 1-2 минуты на человека
```

**После брифинга:**
- Блокеры решаются отдельно (не на брифинге!)
- Все задачи актуализируются в таск-менеджере

---

### 6.3. СОВЕЩАНИЯ

**Отличие от планёрки:** Совещание = обсуждение конкретного вопроса

**Обязательно:**
- **Повестка** сформулирована заранее
- Начинается **вовремя**
- Есть **модератор** и **протоколист**
- **Протокол** выводится на экран (real-time)
- Протокол → рассылается участникам
- Задачи из протокола → таск-менеджер

**Структура протокола совещания:**
- Место проведения (Zoom / офлайн)
- Время проведения
- Тема
- Запись (ссылка)
- Председатель
- Секретарь
- Список присутствовавших
- **Повестка дня** (пронумерованные вопросы)
- **Решения:** № → Решение → Ответственные → Сроки

---

### 6.4. РЕТРОСПЕКТИВА (ежемесячная)

**Цель:** Проанализировать итоги за месяц, поставить планы на следующий

**Последовательность:**
1. **Отчёт по цифровым показателям** (30 мин)
2. **Отчёт по выполненным задачам месяца** (60 мин)
3. **Планы на месяц по задачам развития** (60 мин)
4. **Планы по ключевым метрикам** (30 мин)

**Выход:** Планы на месяц → бэклог → "На месяц" в Kanban

## 7. ФОРМА ПЛАН-ОТЧЁТА

### Структура Google-таблицы

**Каждый участник ведёт свой файл**  
**Каждую неделю — новая вкладка** (название = дата планёрки)

---

### Раздел 1: Отчёт по показателям

| Показатель | Значение | Плановое | Причина отклонения |
|------------|----------|----------|-------------------|
| Количество лидов | 45 | 50 | Снизилась активность в соцсетях |
| Конверсия лид→сделка | 28% | 30% | 2 крупных лида не закрылись |
| Выручка | 2,3M | 2,5M | Перенос оплаты на следующую неделю |

---

### Раздел 2: Отчёт за предыдущую неделю

| План. дата | Задача (по формуле делегирования) | Статус | Результат | Факт дата |
|------------|-----------------------------------|---------|-----------|-----------|

## 8. СИСТЕМА ОКК (Отдел Контроля Качества)

### Определение

**ОКК** = система фиксации и устранения нарушений (косяков) в компании

**Цель:** 
- Не наказывать, а улучшать процессы
- Превращать косяки в точки роста
- Снижать количество нарушений до минимума

### Элементы ОКК

1. **Точки роста** — список типичных косяков
2. **Журнал роста** — фиксация нарушений
3. **Балльная система** — мотивация на качество
4. **Ретроспективы** — анализ косяков и улучшения

### Типичные косяки (примеры точек роста)

**Проектное управление:**
- Задача не записана в таск-менеджере
- Задача без срока или ответственного
- Задача сформулирована НЕ по формуле делегирования
- Опоздание на планёрку
- Не заполнена форма план-отчёта
- Протокол совещания не отправлен в течение 24 часов

**Коммуникации:**
- Не ответил на сообщение в течение 4 часов (рабочее время)
- Грубость с клиентом или коллегой
- Обещал — не сделал без уведомления

**Процессы:**
- Нарушение регламента (любого)
- Не обновил статус задачи в таск-менеджере
- Пропустил дедлайн без уведомления заранее

### Как работает ОКК

**1. Фиксация:**
- Любой сотрудник может зафиксировать косяк
- Форма фиксации: Что? Кто? Когда? Последствия?

**2. Анализ:**
- ОКК Контролёр анализирует системные проблемы
- Разделение: системная проблема vs человеческий фактор

**3. Решение:**
- Если системная проблема → изменить процесс
- Если человеческий фактор → обучение/коучинг
- Критичные нарушения → эскалация собственнику

**4. Мотивация:**
- Балльная система (бонусы зависят от качества)
- Публичный рейтинг качества (опционально)

## 9. ТИПЫ КОМАНД

### Проектная команда

**Определение:** Формируется под конечный результат с этапами и финишной точкой

**Примеры:**
- Строительство дома
- Создание сайта
- Выход на новый рынок

**Ритуалы:**
- Недельная планёрка (обязательно)
- Daily meeting (обычно не нужен!)
- Ретроспектива после завершения проекта

---

### Процессная команда

**Определение:** Формируется под постоянный набор задач, которые не заканчиваются

**Примеры:**
- Отдел продаж
- Отдел производства
- Поддержка клиентов

**Ритуалы:**
- Ежедневные брифинги (ОБЯЗАТЕЛЬНО!)
- Недельная планёрка
- Месячная ретроспектива

---

**Обеим типам нужно проектное управление, но с разными инструментами!**

## 10. ЭТАПЫ ВНЕДРЕНИЯ ПРОЕКТНОГО УПРАВЛЕНИЯ

### Последовательность внедрения (13 шагов)

1. **Провести анализ** существующей системы (сколько команд, какие задачи)
2. **Выбрать таск-менеджер** (Битрикс24, Trello, Asana, etc)
3. **Выгрузить базовый список задач** в бэклог (без подключения команды!)
4. **Продать идею команде** через обучение команды
5. **Разработать правила** для всех элементов проектного управления:
   - Правила ведения таск-менеджера
   - Правила проведения ежедневных брифингов
   - Правила проведения еженедельных планёрок
6. **Обучить команду** работе в таск-менеджере
7. **Выгрузить задачи команды** в бэклог
8. **Перенести задачи** в таск-менеджер
9. **Расставить приоритеты** и ответственных
10. **Внедрить еженедельные планирования**
11. **Внедрить брифинги**
12. **Внедрить ретроспективы**
13. **Внедрить системные совещания**
14. **Внедрить ОКК** на таск-менеджер

**КРИТИЧНО:** Внедрять поэтапно, не всё сразу!

---

### 5 причин, из-за которых не приживается проектное управление

1. **Не продали идею** команде
2. **Не назначили ответственного** за внедрение
3. **Не утвердили правила** с командой
4. **Сами не соблюдаете** правила (подаёте плохой пример)
5. **Поощряете балаган** (не пресекаете несоблюдение правил)

## 11. РАБОТА С АССИСТЕНТОМ (для агента Personal Assistant)

### Система передачи дел

**❌ НЕ скидывать сразу 100+ дел** — сломаете человека!

**✅ Правильно:**
1. Прощупать потихоньку
2. Дать задачи на первую неделю
3. Идеальный расклад: задача на неделю → сделана за 1-2 дня → "Что дальше?"

### Примеры заданий на первую неделю

- Разобрать папку на Google Drive
- Синхронизировать все дела через Google-календарь
- Изучить информацию о компании и конкурентах
- Структурировать информацию о подрядчиках

### Документ "Как работать со мной"

**Обязательные элементы:**

1. **Правила коммуникации:**
   - Каждый вечер уточнять статусы по задачам
   - Подтверждать получение задач
   - К каждой задаче прикладывать нужную информацию
   - По каждой задаче уточнять, если непонятно

2. **Удобные способы коммуникации:**
   - Как удобно получать сообщения (Telegram/Email/Звонок)
   - В какое время нельзя писать
   - Что делать, если не отвечает

3. **Правила переписки:**
   - Все мысли — одним сообщением (не дробить!)
   - Правила звонков

4. **Персональные предпочтения:**
   - Правила бронирования (класс перелёта, тип кровати, etc)
   - Правило планирования (созвоны не раньше 11:00)

---

### Ошибки при работе с ассистентом

**❌ КРИТИЧЕСКИЕ ОШИБКИ СОБСТВЕННИКА:**

1. **"Она должна была догадаться"** — ассистент НЕ телепат!
2. **"Мне нужны тысячи отчётов"** — упрощайте, не создавайте бюрократию
3. **"Написала — молодец, как-нибудь прочитаю"** — покажите внимание
4. **Тон общения** — относитесь как к партнёру, не грубите
5. **"Пусть сами разберутся"** — без чётких ЦУ не будет результата
6. **"Сегодня не будем на планирование"** — нарушение ритуалов = хаос
7. **Отсутствие прозрачного функционала** — нужна таблица функционала
8. **Нарушение собственных ритуалов** — если договорились — делаем!

**Золотое правило:**
> "Не мешайте ассистенту помогать вам. Будьте проактивными, создавайте офигенные отношения."

## 12. СВЯЗИ МЕЖДУ АГЕНТАМИ

### Личный Ассистент ↔ Операционный Координатор

- Ассистент делегирует задачи собственника команде **через Ops Координатора**
- Ассистент **НЕ управляет командой** напрямую

### Ассистент ↔ ОКК Контролёр

- Получает информацию о нарушениях, касающихся собственника
- Фиксирует для собственника дайджест по качеству

### Ops Координатор ↔ ОКК Контролёр

- ОКК фиксирует нарушения в ритуалах
- Ops получает дайджест по качеству для управления
- Ops эскалирует системные проблемы

### Ops Координатор → Внедренец-Системщик

- Передаёт задачи развития → нужен регламент
- Координирует создание новых процессов

## 14. КЛЮЧЕВЫЕ ПРИНЦИПЫ АСБ

### Принцип #1: Комплексность
Все элементы внедряются вместе. Таск-менеджер без ритуалов НЕ РАБОТАЕТ!

### Принцип #2: Приоритет координации
Ритуалы управления — это приоритет #1 компании. Ничто не может их отменить.

### Принцип #3: Системность
Проблема в человеке или в системе? Сначала чинить систему!

### Принцип #4: Измеримость
Всё должно быть измеримо. Задачи, результаты, метрики, KPI.

### Принцип #5: Документированность
Всё должно быть записано. Задачи, решения, процессы, правила.

### Принцип #6: Проактивность
Не ждать проблем, а предупреждать их. PDCA-цикл на всё.

### Принцип #7: Автономность
Цель — собственник занимается только стратегией, всё остальное работает без него.
