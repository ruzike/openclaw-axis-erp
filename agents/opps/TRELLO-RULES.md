# TRELLO WORKFLOW - ops agent

## Операции которые ops может выполнять с Trello:

| Операция | Команда | Описание |
|----------|---------|----------|
| Читать карточку | `trello.py read <card_id>` | Показать описание и метаданные |
| Список карточек | `trello.py list <board> <list>` | Все карточки в колонке |
| Показать чеклисты | `trello.py checklist <card_id>` | Все чеклисты карточки |
| Создать чеклист | `trello.py add-checklist <card_id> 'Название' 'Пункт1' 'Пункт2'...` | Создать новый чеклист |
| Отметить пункт | `trello.py toggle-checkitem <card_id> <item_id> complete` | Поставить ✅ |
| Удалить пункт | `trello.py delete-checkitem <card_id> <item_id>` | Удалить пункт |
| Показать пункты | `trello.py show_checklist <card_id> <checklist_id>` | Все пункты с ID |

## Пайплайн проектов (Доска Производство / C1 C2)

**Стандартные колонки:**
1. **Backlog** — очередь задач, ожидающих взятия в работу.
2. **In Progress** — задачи, которые находятся в активной работе.
3. **QC Review** — задачи, отправленные на проверку качества (Quality Control).
4. **Approve** — задачи на финальном согласовании (или требующие правок после ревью).
5. **Done** — выполненные и сданные задачи.

**Правила постановки задач на визуализацию:**
* **ОДИН РАКУРС = ОДНА КАРТОЧКА.** Нельзя объединять несколько ракурсов в одну карточку.
* **ЧЕКЛИСТ КАРТОЧКИ.** Каждая карточка с ракурсом должна обязательно содержать чеклист со следующими пунктами (или эта информация должна быть четко структурирована в описании):
  * Формат (например, 16:9, 4:5, 1:1)
  * Разрешение (например, 4K, 2K, 1080p)
  * Ракурс (описание вида/камеры)
  * Стиль (mood, референсы)
  * Дедлайн (дата и время сдачи)
