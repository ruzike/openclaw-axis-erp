# Коммуникация с Агентами — Main Agent
**Версия:** 1.0 | **Дата:** 2026-02-19

---

## 🛠️ ИНСТРУМЕНТЫ

### 1. `sessions_send(agentId, message)`
**Тип:** Блокирующий вызов
**Использование:**
- Когда нужен ответ до продолжения диалога
- Прямые запросы информации
- Уточнение статуса

**Пример:**
```
sessions_send(agentId='ops', message='Какие задачи на сегодня?')
```

---

### 2. `sessions_spawn(agentId, task, ...)`
**Тип:** Неблокирующий вызов
**Использование:**
- IT-задачи (devops, tech)
- Долгие задачи (требуют времени)
- Не нужно ждать ответа до продолжения

**Пример:**
```
sessions_spawn(agentId='devops', task='Настроить интеграцию Trello → Notion', runTimeoutSeconds=120, label='IT-задача')
```

---

## 🎯 ТИПЫ АГЕНТОВ

### Технические агенты (используем `sessions_spawn`)
- **Tech** — регламенты, SOP, документация
- **DevOps** — инфраструктура, интеграции, мониторинг

**Почему sessions_spawn:**
- Требуют времени
- Не нужно ждать ответа для продолжения диалога
- Можно отслеживать статус через runId

---

### Операционные агенты (используем `sessions_send`)
- **Ops** — координация, задачи, ритуалы
- **QC** — контроль качества, косяки

**Почему sessions_send:**
- Нужно получить ответ до продолжения
- Прямые запросы информации
- Требуют быстрой реакции

---

### Люди (используем `sessions_send`)
- **HR** — вопросы людей, обучение, адаптация
- **Finance** — финансовые вопросы, отчёты
- **Sales** — вопросы продаж, клиенты

**Почему sessions_send:**
- Прямое общение с людьми
- Требуют быстрой реакции
- Контекст важен (личные вопросы)

## 📋 ПРАВИЛА КОММУНИКАЦИИ

### 1. Не спамить в общий чат

**Правило:**
- Если мне нужно узнать статус у Ops — я пишу ему через `sessions_send`
- Ops сам сообщает в общий чат если это важно

**Пример:**
- ✅ `sessions_send(agentId='ops', message='Статус задач на сегодня?')`
- ❌ Писать в общий чат: "Ops, как дела?"

---

### 2. Всегда указывать контекст

**Правило:**
- В сообщении всегда указываю контекст задачи
- Агент не имеет истории моего текущего диалога

**Пример:**
- ✅ `sessions_send(agentId='ops', message='Руслан просит узнать статус задачи по регламенту C4')`
- ❌ `sessions_send(agentId='ops', message='Какие задачи?')`

---

### 3. Для tech/devops — только sessions_spawn

**Правило:**
- Никогда не использую `sessions_send` для технических задач
- Использую `sessions_spawn(agentId='devops', task='...', runTimeoutSeconds=120)`

**Пример:**
- ✅ `sessions_spawn(agentId='devops', task='Проверить статус серверов', runTimeoutSeconds=60, label='IT-задача')`
- ❌ `sessions_send(agentId='devops', message='Проверь серверы')`

---

### 4. Жду ответа после sessions_send

**Правило:**
- После `sessions_send` я жду ответа от агента
- Не продолжаю диалог пока не получу ответ

**Пример:**
```
sessions_send(agentId='ops', message='Какие задачи на сегодня?')
→ Жду ответ от Ops

→ После получения ответа
→ Продолжаю диалог
```

---

### 5. Проверяю статус после sessions_spawn

**Правило:**
- После `sessions_spawn` проверяю статус через runId
- Ответ агент пришлёт сам

**Пример:**
```
sessions_spawn(agentId='devops', task='Настроить интеграцию', runTimeoutSeconds=120, label='IT-задача')
→ Сохраняю runId
→ Через 120 секунд проверяю статус
→ Если completed — получаю результат
```

## 📊 ПРИМЕРЫ ВЫЗОВОВ

### Пример 1: Узнать статус задачи
```python
# Опрашиваем Ops
sessions_send(agentId='ops', message='Руслан просит узнать статус задачи по регламенту C4')
# Ops отвечает в общем чате: "Задача выполнена"

# Сообщаем Руслану
"Задача по регламенту C4 выполнена"
```

---

### Пример 2: Сделать задачу
```python
# Делегируем DevOps
sessions_spawn(
    agentId='devops',
    task='Настроить интеграцию Trello → Notion для отчётов по регламенту',
    runTimeoutSeconds=120,
    label='IT-задача'
)
# Сохраняю runId: 'abc123'

# Жду 120 секунд
# Проверяю статус
# Если completed — получаю результат
```

---

### Пример 3: Создать регламент
```python
# Делегируем Tech
sessions_spawn(
    agentId='tech',
    task='Создать регламент C6 "Ведение клиентов" и внедрить его в систему',
    runTimeoutSeconds=120,
    label='Регламент/внедрение'
)
# Сохраняю runId: 'def456'

# Жду 120 секунд
# Проверяю статус
# Если completed — получаю результат
```

## 📋 ЧЕК-ЛИСТ ПЕРЕД ВЫЗОВОМ

Перед вызовом агента проверяю:

- [ ] **Тип агента:** Технический (sessions_spawn) или операционный/людской (sessions_send)?
- [ ] **Контекст:** Понятно что нужно сделать?
- [ ] **Срок:** Если есть дедлайн, указан ли он?
- [ ] **Контроль:** В каком виде нужен отчёт?

**Если 4/4 — вызываю**
**Если 0-3/4 — уточняю контекст**
