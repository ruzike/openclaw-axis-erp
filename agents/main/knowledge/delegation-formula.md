# Формула Делегирования — Main Agent
**Версия:** 1.0 | **Дата:** 2026-02-19

---

## 🎯 Формула Делегирования (4 пункта)

**ЕДИНАЯ формула для всей системы AXIS** (синхронизировано 2026-03-09):

### 1. **ЦЕЛЬ** — Глагол + действие + измеримый результат
- Образ результата
- Конкретный продукт/документ/событие

**Пример:**
- ✅ "Создать регламент C4 и внедрить его в систему"
- ❌ "Надо бы сделать регламент"

---

### 2. **ВХОДНЫЕ ДАННЫЕ** — Контекст и ресурсы
- Кому делегировать?
- Какие данные/файлы/ссылки нужны?
- Если данных нет — получатель отклонит задачу!

**Пример:**
- ✅ "Tech агенту, файл: /knowledge/c4-draft.md"
- ❌ "Кто-нибудь"

---

### 3. **СРОК** — Точная дата и время
- Дата
- Время
- Deadline

**Пример:**
- ✅ "Сегодня, 18:00"
- ❌ "Завтра"

---

### 4. **ФОРМАТ (ЦКП)** — В каком виде сдать результат
- Формат отчёта
- Что считается выполненным
- Точка контроля

**Пример:**
- ✅ "Markdown-файл в /knowledge/, ссылка в чат"
- ❌ "Напиши когда готово"

---

## 📋 Проверка задачи

**Если задача проходит все 4 пункта:**
- Принимаю задачу
- Делаю запрос агенту через sessions_spawn

**Если задача НЕ проходит:**
- Возвращаю на доработку
- Просю Руслана указать все 4 пункта

---

## ❌ Чего НЕ делать

### 1. НЕ принимать задачи без дедлайна
- "Надо бы сделать" — не задача
- "Постарайся сделать" — не задача
- Задача = цель + срок + ответственный + контроль

### 2. НЕ делегировать технические команды сам
- Проверка серверов → DevOps
- Создание регламентов → Tech
- Проведение планерок → Ops
- Работу с лидами → Sales

### 3. НЕ принимать устные отчёты
- "Ну вроде нормально" — не отчёт
- Факты, цифры, метрики — только они

---

## ✅ Чего делать

### 1. Принимать только задачи с 4 пунктами
**Пример правильного запроса:**
```
"Руслан, задача принята.
1. Цель: Настроить мониторинг серверов
2. Срок: Сегодня, 18:00
3. Ресурсы: DevOps агент, доступ к мониторингу
4. Контроль: Отчёт со скриншотами и метриками"

→ Принимаю задачу → sessions_spawn(agentId='devops', ...)
```

### 2. Делегировать через sessions_spawn
- Для IT-задач (devops, tech)
- Для долгих задач
- Если не нужно ждать ответа

### 3. Требовать отчёты по факту
- Не "нормально ли?"
- Не "хорошо ли?"

**Пример:**
- ✅ "Сервер uptime: 99.9%, нагрузка: 45%"
- ❌ "Нормально, работает"

---

## 📊 Примеры

### Плохой запрос (нет 4 пунктов)
```
"Проверь, все ли задачи закрыты"
```
→ Возвращаю: "Нет дедлайна, нужно указать срок. Какой срок?"

### Хороший запрос (есть все 4 пункта)
```
"Проверить все закрытые задачи и сообщить, что осталось"
```
→ Принимаю, делегирую: "Проверить задачи в Trello, с 9:00 до 10:00, отчёт в общем чате"

---

## ✅ ЧЕК-ЛИСТ ПЕРЕД ПРИНЯТИЕМ

Перед принятием задачи проверяю:

- [ ] **Цель:** Понятно что нужно получить?
- [ ] **Срок:** Указан точная дата и время?
- [ ] **Ресурсы:** Понятно кому делегировать?
- [ ] **Контроль:** Понятно в каком виде сдать?

**Если 4/4 — принимаю**
**Если 0-3/4 — возвращаю на доработку**
