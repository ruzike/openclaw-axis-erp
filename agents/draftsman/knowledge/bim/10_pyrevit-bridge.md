# pyRevit Bridge — Инструкция для агента

> Мост между агентом и Revit через файл команд JSONL.
> Агент пишет команды → bridge выполняет в Revit автоматически.

---

## КАК РАБОТАЕТ МОСТ

```
Агент получает задачу
        ↓
Генерирует JSONL команды
        ↓
Сохраняет в C:\bridge\revit_commands.jsonl
        ↓
Мирас нажимает кнопку "BuildFromJSON" в Revit
        ↓
script.py выполняет все команды в Revit
        ↓
Файл очищается автоматически после выполнения
```

**Путь к файлу команд:** `C:\bridge\revit_commands.jsonl`
**Кнопка запуска:** MyTab → MyPanel → BuildFromJSON

---

## ЕДИНИЦЫ ИЗМЕРЕНИЯ

**ВСЕГДА миллиметры.** Script.py конвертирует мм → футы автоматически через функцию `ft(mm)`.

```
1000 мм = 1 метр
300 мм = толщина стены
2800 мм = высота стены по умолчанию
```

---

## ФОРМАТ КОМАНД (JSONL)

Каждая команда — отдельная строка JSON. Порядок важен:
1. Сначала `level` (уровни)
2. Потом `grid` (оси)
3. Потом `wall` (стены)
4. Потом `door` и `window` (проёмы — только после стен)

---

## КОМАНДЫ

### level — Создать уровень
```json
{"command": "level", "name": "1 этаж", "elevation_mm": 0}
{"command": "level", "name": "2 этаж", "elevation_mm": 2800}
```

| Параметр | Тип | Описание |
|---|---|---|
| name | string | Имя уровня |
| elevation_mm | number | Высота в мм от нуля |

---

### grid — Создать ось
```json
{"command": "grid", "name": "1", "x1_mm": 0, "y1_mm": -500, "x2_mm": 0, "y2_mm": 8000}
{"command": "grid", "name": "А", "x1_mm": -500, "y1_mm": 0, "x2_mm": 10000, "y2_mm": 0}
```

| Параметр | Тип | Описание |
|---|---|---|
| name | string | Имя оси (цифра или буква) |
| x1_mm, y1_mm | number | Начало оси |
| x2_mm, y2_mm | number | Конец оси |

**Правило:** Вертикальные оси — меняется X, Y одинаковый. Горизонтальные — меняется Y, X одинаковый.

---

### wall — Создать стену
```json
{"command": "wall", "x1_mm": 0, "y1_mm": 0, "x2_mm": 5000, "y2_mm": 0, "height_mm": 2800, "thickness_mm": 300, "level": "1 этаж"}
{"command": "wall", "x1_mm": 0, "y1_mm": 0, "x2_mm": 0, "y2_mm": 4000, "height_mm": 2800, "thickness_mm": 200, "level": "1 этаж"}
```

| Параметр | Тип | Описание |
|---|---|---|
| x1_mm, y1_mm | number | Начало стены |
| x2_mm, y2_mm | number | Конец стены |
| height_mm | number | Высота стены (обычно 2800) |
| thickness_mm | number | Толщина стены в мм |
| level | string | Имя уровня (из `08_template-structure.md`) |

**Толщины стен по стандарту AXIS (`01_naming-standard.md`):**
| Тип стены | thickness_mm |
|---|---|
| Monolith 300 | 300 |
| Monolith 200 | 200 |
| Кирпич 250 | 250 |
| Блок 300 | 300 |
| ГКЛ 75 | 75 |
| НГКЛ 100 | 100 |

---

### door — Вставить дверь
```json
{"command": "door", "x_mm": 2500, "y_mm": 0, "level": "1 этаж"}
```

| Параметр | Тип | Описание |
|---|---|---|
| x_mm, y_mm | number | Центр двери на стене |
| level | string | Имя уровня |

**Важно:** Дверь ищет ближайшую стену автоматически. Координаты должны быть точно на линии стены.

---

### window — Вставить окно
```json
{"command": "window", "x_mm": 1500, "y_mm": 0, "sill_mm": 900, "level": "1 этаж"}
```

| Параметр | Тип | Описание |
|---|---|---|
| x_mm, y_mm | number | Центр окна на стене |
| sill_mm | number | Высота подоконника от пола (обычно 900) |
| level | string | Имя уровня |

## АЛГОРИТМ РАБОТЫ АГЕНТА

Когда агент получает задачу построить план:

1. **Читает** `08_template-structure.md` — проверяет уровни
2. **Уточняет** размеры помещений у пользователя (если не указаны)
3. **Генерирует** JSONL команды в правильном порядке:
   - Сначала уровни
   - Потом оси
   - Потом несущие стены (thickness 300)
   - Потом перегородки (thickness 75-200)
   - Потом двери и окна
4. **Выводит** готовый JSONL блок пользователю
5. **Говорит:** "Скопируй в `C:\bridge\revit_commands.jsonl` и нажми BuildFromJSON"

---

## ПРАВИЛА И ОГРАНИЧЕНИЯ

- ❌ Нельзя вставить дверь/окно до создания стены-хоста
- ❌ Нельзя создать стену нулевой длины (x1=x2 и y1=y2)
- ❌ Уровень в команде должен точно совпадать с именем уровня в модели
- ✅ После выполнения файл очищается автоматически — можно писать новые команды
- ✅ Координаты двери/окна — примерно на линии стены, bridge найдёт хост сам
- ✅ Все размеры в мм — никаких футов и метров в командах
