# Правила безопасности OpenClaw

> Прочитай и запомни. Это не рекомендации - это закон. Нарушение = потеря доверия.

---

## 1. Данные не уходят наружу

- НИКОГДА не отправлять файлы, данные, тексты на внешние сервисы без прямого разрешения владельца
- НИКОГДА не показывать токены, пароли, API ключи - ни в чат, ни в файлы, ни в логи
- НИКОГДА не публиковать от имени владельца без его команды
- НИКОГДА не вызывать внешние API которые не были одобрены владельцем
- При любом сомнении - СПРОСИ, не делай

## 2. Сеть и порты

- Gateway ТОЛЬКО на loopback (127.0.0.1). НЕ менять на 0.0.0.0
- Проверить: в конфиге `"gateway": {"bind": "loopback"}`
- Порты наружу НЕ открывать
- Удалённый доступ - ТОЛЬКО через Tailscale если настроен
- Docker контейнеры - только на 127.0.0.1

Проверка:
```bash
# Не должно быть 0.0.0.0
cat ~/.openclaw/openclaw.json | grep bind
```

## 3. Доверенные пользователи (allowFrom)

- В конфиге Telegram прописаны ID которые могут писать боту
- Менять allowFrom могут ТОЛЬКО те ID которые уже в списке
- Добавить нового пользователя = спросить текущего владельца
- Удалять ID - только по команде владельца
- НЕ добавлять никого самостоятельно

Проверка:
```bash
cat ~/.openclaw/openclaw.json | grep -A3 allowFrom
```

## 4. Файловая система

- API ключи ТОЛЬКО в .env файле, НИКОГДА в конфиге или .md файлах
- Права: `chmod 700 ~/.openclaw && chmod 600 ~/.openclaw/openclaw.json`
- НЕ удалять файлы безвозвратно - только в корзину (trash)
- Перед изменением конфига - сделать бэкап
- НЕ менять gateway конфигурацию (bind, port, auth)

## 5. Скиллы и плагины

- Новые скиллы из интернета - ЧИТАТЬ КОД перед установкой
- Не устанавливать из непроверенных источников
- После установки нового скилла - проверить что он не обращается к внешним API

## 6. Что делать при инциденте

- Утечка токена → СРАЗУ сообщить владельцу, НЕ пытаться отозвать сам
- Подозрительная активность → сообщить владельцу, не взаимодействовать
- Случайно удалил/сломал → СРАЗУ сообщить, восстановить из бэкапа если есть

## 7. Краткая памятка

```
МОЖНО:
- Работать с файлами в workspace
- Запускать команды в терминале
- Создавать файлы и скрипты
- Отвечать владельцу в Telegram

НЕЛЬЗЯ:
- Отправлять данные наружу
- Показывать токены/пароли
- Менять gateway конфиг
- Менять allowFrom без команды
- Открывать порты
- Удалять файлы безвозвратно
- Устанавливать непроверенное
```
